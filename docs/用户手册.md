# 用户手册

autodo-engine 提供运行时初始化、图注册、任务推进、审计查询与事务直调能力。官方事务来源于已安装的 autodo-kit。

## 1. 安装

```powershell
uv pip install -e .
uv pip install -e ../autodo-kit
```

## 2. 初始化运行时

```powershell
python -m autodoengine.main init-runtime --base-dir ./.tmp/runtime_v4
```

## 3. 注册流程图并创建任务

```powershell
python -m autodoengine.main register-graph --graph-file workflows/workflow_20260308_all_affairs_demo/workflow.json
python -m autodoengine.main create-task --title "示例任务" --goal-text "验证运行链路" --current-node-uid node-start
```

## 4. 推进任务

```powershell
python -m autodoengine.main run-task-step --task-uid <task_uid> --graph-uid <graph_uid>
python -m autodoengine.main run-task --task-uid <task_uid> --graph-uid <graph_uid> --max-steps 50
```

## 5. 查看审计与数据库路径

```powershell
python -m autodoengine.main show-task --task-uid <task_uid>
python -m autodoengine.main show-decisions --task-uid <task_uid>
python -m autodoengine.main show-runtime-events --task-uid <task_uid>
python -m autodoengine.main show-runtime-store-paths --base-dir ./.tmp/runtime_v4
python -m autodoengine.main show-affair-registry-paths --workspace-root .
```

## 6. 直接调用事务

```python
from pathlib import Path
from autodoengine import api

api.run_affair(
    "Word转LaTeX",
    config={"input_docx": "data/demo.docx", "output_dir": "output/demo"},
    workspace_root=Path.cwd(),
)
```

说明：

- 事务解析、冲突检测、用户覆盖规则在本仓执行。
- 官方事务源码与事务文档来自 autodo-kit。
- 若未安装 autodo-kit，事务直调与工具查询接口不会生效。
