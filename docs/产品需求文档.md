# 产品需求文档

## 1. 定位

autodo-engine 是通用任务事务工作流系统的运行时引擎，负责把流程图、任务状态、候选动作、决策结果和审计记录组织成可执行闭环。同时还负责各类数据库的访问与维护、流程图的加载与调度、事务库的注册与发现等核心功能。它和 autodo-kit 共同构成了 AOK 的核心框架层，提供给上层业务开发者一个稳定的运行时环境和事务内容库。

## 2. 本阶段目标

1. 保持核心运行时以 autodoengine 包独立交付。
2. 让流程图执行、任务数据库、决策闭环与审计视图形成稳定主链路。
3. 通过事务注册系统引用外部事务库，而不是把事务实现重新耦合回引擎仓。

## 3. In Scope

- core / flow_graph / scheduling / taskdb / utils
- Python API 与 CLI
- 事务注册、冲突校验、运行时合并视图
- 决策部门、阻断治理、审计视图

## 4. Out of Scope

- 官方事务实现
- 事务手册
- 事务侧工具集合

## 5. 关键约束

1. 引擎仓不内置官方 affairs 目录。
2. 官方事务从 autodo-kit 发现；用户事务从工作区 .autodokit 发现。
3. 包名、文档与命令全部统一使用 autodoengine 口径。
